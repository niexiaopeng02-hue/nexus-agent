# Video Recording Checklist

## Before Recording

- [ ] Use a 1440x900 or 1920x1080 browser window.
- [ ] Close bookmarks bar and unrelated tabs.
- [ ] Clear stale demo/test data or start from a fresh seeded database.
- [ ] Confirm network stability.
- [ ] Confirm frontend has no visible console/runtime errors.
- [ ] Confirm fonts render normally.
- [ ] Prepare demo questions:
  - `What is NovaTech's return policy?`
  - `Where is order ORD-10001?`
  - `My keyboard stopped working. Please create a support ticket.`
  - `Do you offer drone insurance?`
- [ ] Check microphone and screen recording audio.
- [ ] Keep secrets and local terminal paths out of the recording.

## Recording Flow

- [ ] 0-10s: GitHub or landing page intro.
- [ ] 10-25s: RAG answer with citation.
- [ ] 25-40s: Order tool call.
- [ ] 40-55s: Ticket creation.
- [ ] 55-70s: No-context refusal.
- [ ] 70-90s: Knowledge Base, Analytics, architecture, GitHub, live demo URL if deployed.

## After Recording

- [ ] Verify text is readable.
- [ ] Verify no secrets are visible.
- [ ] Export 1080p video.
- [ ] Upload to portfolio platform or unlisted video hosting.
- [ ] Add the real video URL to README only after upload.
